Introspect — MVP Plan

A journaling app that learns your habits and nudges you toward intentional action.

1 Core User Flow

Step 1

📝 Write an Entry

Open the app every few hours. Free-write what you did, what you're thinking, and your plan.

→
Step 2

🤖 AI Analyzes

Gemini reads your entries via Vercel AI SDK, detects recurring habits, time patterns, and sentiment.

→
Step 3

📊 See Insights

Dashboard shows detected habits, streaks, and how your time is being spent.

→
Step 4

⚡ 2-Min Action

Get one concrete thing you can do right now to stay aligned with your goals.

2 How It All Fits Together

3 Tech Stack (MVP)

Scaffolding

create-t3-app

Next.js + Drizzle + Tailwind + TypeScript in one command. Swap the default Postgres adapter for SQLite via better-sqlite3.

Styling

Tailwind CSS v4

Rapid UI development with utility classes. Pairs perfectly with Next.js out of the box.

Database

SQLite (local) → Turso (prod)

Local dev uses better-sqlite3, zero infrastructure. Deploy swaps to Turso (SQLite-compatible, free tier, one-line Drizzle adapter change).

ORM

Drizzle ORM

Lightweight, type-safe ORM that works great with SQLite. Simple migrations.

AI Model

Gemini

Free API key from aistudio.google.com. Fast, cheap, handles habit extraction, insight generation, and 2-min nudges.

AI SDK

Vercel AI SDK

npm install ai @ai-sdk/google. Abstracts Gemini calls, handles streaming and structured JSON parsing out of the box.

Deploy

Vercel

One-click deploy. Production DB is Turso (swap better-sqlite3 → @libsql/client in Drizzle config). Free tier covers MVP.

Why this stack? — create-t3-app scaffolds the entire project in one command (Next.js + Drizzle + Tailwind + TS). Gemini is free via AI Studio. Vercel AI SDK abstracts the LLM calls. SQLite locally, Turso in prod — same Drizzle schema, one adapter swap. Everything runs with npm run dev.

5 AI Prompt Strategy (2 calls)

Call 1 — On Entry Save

Extract Habits

Send the new entry + last 5 entries to Gemini via Vercel AI SDK. Use generateObject() for structured JSON: detected habits (name, sentiment) and a single 2-min nudge.

→
Call 2 — On Dashboard Load

Generate Insights

Send the habits table to Gemini. Use generateText() for a short summary: what's going well, what's slipping, overall trajectory.

Keep it cheap. Only 2 Gemini calls in the whole app. Vercel AI SDK's generateObject() gives us Zod-validated JSON — no manual parsing. No embeddings, no vector DB — that's for later.

7 Build Order

  • Phase 1: Run create-t3-app, swap Postgres → SQLite (better-sqlite3), define the 3 Drizzle tables.
  • Phase 2: Build the Journal page — textarea → save to DB → show past entries.
  • Phase 3: Wire up Gemini via Vercel AI SDK — extract habits + generate nudge on entry save.
  • Phase 4: Build the Insights page — show habits, streaks, AI summary.
  • Phase 5: Polish — styling, responsive layout, loading states, error handling.
Estimated time: ~2-3 focused sessions to get a working MVP. Each phase is independently testable.
